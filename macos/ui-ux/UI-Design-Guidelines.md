# macOS UI 设计规范（当前实现基线）

更新时间：2026-04-29  
适用范围：YakType macOS（首页、引擎编排、引擎页）

## 1. 设计目标
- 与 macOS 原生视觉一致，减少自定义装饰感。
- 在深色/浅色模式下使用系统语义色，不做额外主题偏色。
- 保持信息分区清晰：左侧主任务，右侧辅助信息。

## 2. 页面布局基线

### 2.1 首页（Home）
- 使用双栏布局。
- 左右列宽比例：`58 : 42`（6:4基础上右侧微增）。
- 左列：最新识别、使用趋势。
- 右列：活动快捷键（从引擎编排实际配置提取）。

### 2.2 引擎编排（Engine Orchestration）
- 使用双栏布局。
- 左右列宽比例：`58 : 42`。
- 左列：听写引擎、后处理节点。
- 右列：连通性检测。
- Section 结构统一为：标题（handle） + 内容卡片，不使用额外 icon 前缀。

### 2.3 引擎页（Engine Management）
- 结构：左侧原生列表 + 右侧 Inspector 详情。
- 左侧使用 `List + Section`，风格采用 `insetGrouped`。
- 右侧详情固定为 Inspector 展示，未选中时显示占位说明。

## 3. 视觉规范

### 3.1 卡片
- 所有业务卡片取消阴影。
- 通过以下方式建立层级：
  - 圆角
  - 系统背景色（`controlBackgroundColor` / 语义面色）
  - 轻边框（separator 低透明度）

### 3.2 颜色
- 全站优先使用系统语义色：`primary / secondary / separator / controlBackgroundColor`。
- 深浅色自动切换，避免手写硬编码黑白背景。

### 3.3 工具栏
- 采用统一的顶部渐隐透明材质效果（与其他页面一致）。
- 工具项按功能分组：新增、筛选/分段、删除/克隆等。

## 4. 交互规范
- 列表选中项驱动右侧详情；点击列表空白可清空选中。
- 引擎与密钥编辑优先就地编辑（Inspector 内），减少弹窗。
- 活动快捷键展示必须反映引擎编排当前配置，而非 iOS 手势语义。

## 5. 当前已落地项（本次）
- 全局移除卡片阴影（包含首页/引擎编排/快捷键卡片容器相关组件）。
- 引擎页列表改为原生 `insetGrouped` 风格，行文字层级改为系统常用层级。
- 引擎页 Inspector 宽度收敛为更接近系统设置类页面的范围。

## 6. 后续建议
- 将引擎详情编辑行进一步统一为 `List + Section + LabeledContent`（完全原生表单感）。
- 把页面级间距与标题字号抽出统一 token，减少页面内局部硬编码。
