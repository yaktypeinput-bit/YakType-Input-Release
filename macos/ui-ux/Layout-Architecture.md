# macOS 局部布局架构 (Layout Architecture)

## 概要说明

本文档定义了 YakType 应用的基础空间布局模型。应用采用了层次分明的“浮动组件（Floating Components）”设计理念，旨在 macOS 平台上营造出轻量、通透且具备深度的视觉体验。通过理解本架构，开发者可以掌握如何正确放置新的 UI 模块，并保持各层级间的逻辑一致性。

## 全局布局骨架

应用的主界面由四个核心区域构成，采用非对称的响应式布局：

```text
+-----------------------------------------------------------+
| [Traffic Lights]                                [Header]  |
|                                                           |
| +----------+  +-----------------------------------------+ |
| | Sidebar  |  | Main Content Area                       | |
| | (Fixed)  |  | (Responsive)                            | |
| |          |  |                                         | |
| | [Home]   |  |  +-----------------------------------+  | |
| | [History]|  |  |                                   |  | |
| | [Prompts]|  |  |        [Page Content View]        |  | |
| | [Engines]|  |  |                                   |  | |
| | [Settings]|  |  +-----------------------------------+  | |
| |          |  |                                         | |
| | [Logo]   |  |          [Bottom Toolbars]              | |
| +----------+  +-----------------------------------------+ |
+-----------------------------------------------------------+
```

1.  **左侧导航 (Floating Sidebar)**：宽度固定为 240pt，整体浮动于背景层之上，通过 `appSurface` 材质实现层级切分。
2.  **顶部状态栏 (Header)**：位于内容区右上角，显示实时引擎、音频源、处理状态及版本号。
3.  **主内容区 (Main Content)**：承载各功能模块的主视图。
4.  **背景层 (Tiled Background)**：位于最底层，负责渲染动态生成的液体光晕与全局模糊效果。

## 层级管理 (Layering & Z-Index)

应用通过 `ZStack` 实现精密的视觉堆叠：

| 层级 | 深度 (Z-Index) | 内容 | 视觉处理 |
| :--- | :--- | :--- | :--- |
| **背景层** | 0 | `TiledBackground` | 动态模糊、液体光晕、主背景色适配。 |
| **内容层** | 10 | 各功能主视图 (`Home`, `Tasks` 等) | 具备内边距，确保内容不被侧边栏遮挡。 |
| **详情层** | 20 | `FloatingDetailSplitView` 的详情项 | 带有柔和阴影（`ThemeShadow.floating`）。 |
| **导航层** | 30 | `SidebarView`, `HeaderView` | `.base` 级别的玻璃材质。 |
| **模态层** | 100 | `HUD`, `Onboarding`, `Config Modal` | 全局模糊背景（Blur Radius: 10），模态居中。 |

## 响应式策略 (Responsiveness)

*   **窗口尺寸**：最小宽度限制为 1200pt，最小高度为 650pt。
*   **侧边栏固定**：侧边栏始终保持在左侧，不随内容滚动，确保导航入口永远可见。
*   **内容弹性**：主内容区与详情卡片采用弹性宽度（`max(minDetailWidth, availableWidth)`），在大屏幕上自动展开，在标准屏幕上保持紧凑布局。
