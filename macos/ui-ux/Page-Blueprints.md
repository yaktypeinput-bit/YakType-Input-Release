# macOS 页面蓝图与原型 (Page Blueprints)

## 概要说明

本文档详述了 YakType 各核心页面的详细设计蓝图。通过 ASCII 原型图与技术说明，展示了各功能模块内部的组件分布、交互热区以及特有的设计模式，为后续的功能扩展或 UI 迭代提供精准的设计参考。

---

## 1. 任务历史记录 (Tasks List / History)

采用“列表 + 悬浮详情”的双栏布局。

```text
+-----------------------------------------------------------+
| [Recent Activities]                                       |
|                                                           |
| +---------------------+  +------------------------------+ |
| | [Task Item Card 1]  |  | [Detail View Header]         | |
| | - Date / Status     |  |                              | |
| | - Preview Snippet   |  | +--------------------------+ | |
| | [Task Item Card 2]  |  | | [Dictation / Polished]   | | |
| |                     |  | | [Result Text Area    ]   | | |
| | [Task Item Card 3]  |  | |                          | | |
| |                     |  | | [Metadata Table      ]   | | |
| |                     |  | | [Action Buttons Row  ]   | | |
| +---------------------+  +------------------------------+ |
|                                                           |
| [Toolbar: Add/Search]                                     |
+-----------------------------------------------------------+
```

*   **交互模式**：
    *   **左侧列表**：支持无限滚动加载。每一个卡片具备悬停态（显示复制/删除按钮）。
    *   **右侧详情**：通过 `selectedTaskID` 驱动内容更新。包含一个分段选择器（ASR/后处理），以及音频元数据详情表。
    *   **底部工具栏**：采用 Capsule 风格，动态显示搜索框或功能按钮，并附带阴影光晕。

---

## 2. 引擎管理 (Engine Management)

该页面是系统中交互复杂度最高的部分，强调“流水线化 (Pipeline)”的可视化配置。

```text
+---------------------+      +------------------------------+
| [Engine Instances] |      | [Processing Pipeline]        |
|                     |      |                              |
| +-----------------+ |      | +--------------------------+ |
| | [Gemini Pro  ]  | | ---> | | [ Dictation Step Target ] | |
| +-----------------+ | Drag | +--------------------------+ |
|                     |      |             |                |
| +-----------------+ |      | +--------------------------+ |
| | [Aliyun Flash]  | |      | | [ Polishing Step Target] | |
| +-----------------+ |      | +--------------------------+ |
+---------------------+      +------------------------------+
```

*   **设计模式 (Drag-n-Drop)**：
    *   用户从左侧实例列表（卡片）中拖拽目标，放置到右侧流水线的特定步骤（Target Box）中。
    *   **实时反馈**：放置区在拖拽进入时会显示描边高亮，根据引擎兼容性显示“Accepted”或“Rejected”状态。

## 3. 录制悬浮窗 (Floating HUD)

HUD 是应用的“交互之魂”，采用极致简约的设计。

```text
          (Screen Center)
      +-----------------------------+
      | [X]                         |
      |   [Status: Listening...]    |
      |                             |
      |   [ ~~~~ Waveform ~~~~ ]    |
      |                             |
      |   [Duration: 00:12]         |
      +-----------------------------+
```

*   **视觉规范**：
    *   **状态驱动**：根据不同的处理阶段（Recording, Transcribing, Polishing）动态切换图标与文字提示。
    *   **极简动画**：波形图（Waveform）通过实时振幅驱动高度变化，提供“生命感”反馈。

---

## 4. 提示词管理 (Prompts)

采用分层堆叠的卡片列表，方便用户快速编辑与切换 AI 角色。

*   **布局**：单栏列表。每一项为一个 `PromptTemplate` 卡片，包含标题预览与修改时间。

## 5. 首页仪表盘 (Home Dashboard)

首页采用了“主从分栏 (Primary-Sidebar)”布局，将动态内容整合在左侧宽栏，而将参考信息整合在右侧边栏。

```text
+-----------------------------------------------------------+
| [Left Column | 70%]              | [Right Column | 30%]   |
|                                  |                        |
| +------------------------------+ | +--------------------+ |
| | [Latest Recognition]         | | | [Active Shortcuts] | |
| | - Full Width for Snippet     | | | - Single Column    | |
| | - Action Row                 | | | - List View        | |
| +------------------------------+ | | - High Legibility  | |
|                                  | |                    | |
| +------------------------------+ | | [Expandable Toggle] | |
| | [Usage Trend]                | | +--------------------+ |
| | - Bar Chart                  | |                        |
| | - Stats Row                  | |                        |
| +------------------------------+ |                        |
+-----------------------------------------------------------+
```

*   **布局逻辑**：
    *   **左侧主栏 (Primary Area)**：占宽 70%。
        *   **模块 A (Latest Recognition)**：位于顶部，展示最近任务详情。由于宽度增加，文本展示更为从容。
        *   **模块 B (Usage Trend)**：位于主栏下方，展示时长、字符等统计图表。
    *   **右侧边栏 (Sidebar Area)**：占宽 30%。
        *   **模块 C (Active Shortcuts)**：采用 **“单列列表 (List)”** 展现形式。每一行展示一个完整的快捷键（包含图标、标签、按键），追求极致的查询便利性与易读性。
*   **交互规范与兜底方案**：
    *   **深度自适应 (Vertical Scroll)**：由于右侧边栏可能因快捷键较多而变长，整个页面使用全局 `ScrollView`。左右两栏在顶部对齐，高度随内容自适应。
    *   **动态展示**：快捷键列表依然支持“超出 10 条动态展开”的逻辑，确保在极端数据下不破坏 Dashboard 的整体高度观感。
    *   **视觉引导**：右侧卡片使用统一的 `minHeight`，以便在初始状态下与左侧“最新识别”保持视觉平衡。

## 6. 初始化引导 (Onboarding Wizard)

采用 4 步分阶段模态窗布局，强调引导性与操作稳定性。

```text
+-----------------------------------------------------------+
| [Title: Step Name]                                        |
| [Subtitle: Instructions]                                  |
|                                                           |
| +-------------------------------------------------------+ |
| |                                                       | |
| |                 [ Step Content Area ]                 | |
| |           (Scrollable, Vertical Centered)             | |
| |                                                       | |
| +-------------------------------------------------------+ |
|                                                           |
| [ ~~~~~~~~~~~~~ Gradient Mask Overlay ~~~~~~~~~~~~~ ]     |
| +-------------------------------------------------------+ |
| |           [Step Indicator: O O O O]                   | |
| |                                                       | |
| | [Back Button] (Optional)          [Next Step Button]  | |
| +-------------------------------------------------------+ |
+-----------------------------------------------------------+
```

*   **关键架构**：
    *   **固定底座 (Fixed Footer)**：由 `StepIndicator` 与导航按钮组成，通过 `ZStack` 置于最上层，不随内容滚动而位移。
    *   **内容区 (Scrollable Container)**：支持垂直弹性滚动，确保在低分辨率屏幕下表单内容依然可访问。
    *   **视觉引导**：底部遮罩层（Linear Gradient）提供了深度感，同时也解决了内容遮挡按钮的问题。
