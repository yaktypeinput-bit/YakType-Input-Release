# macOS 悬浮窗与交互逻辑 (HUD Interactions)

## 概要说明

本文档详述了悬浮窗（Floating HUD）的生命周期管理、阶段性视觉反馈机制以及键盘交互设计。作为 YakType 与用户交互频率最高的核心组件，HUD 的稳定性和即时反馈逻辑直接影响到用户的录音体验。

## HUD 生命周期

### 触发机制
1.  **自动触发**：通过全局快捷键启动录音时，HUD 自动弹出并进入 `recording` 状态。
2.  **手动触发**：用户在主界面点击录制按钮。

### 隐藏逻辑
*   **成功完成**：任务进入 `success` 状态后，HUD 保留 **1.25 秒** 供用户确认结果，随后自动淡出。
*   **发生错误**：任务进入 `error` 状态后，HUD 保留 **3.5 秒** 展示错误详情，确保用户有足够时间阅读。
*   **手动关闭**：点击右上角关闭按钮或通过 Esc 键中断。

## 处理阶段反馈

HUD 会根据 `TranscriptionService` 的状态变量进行实时更新：

| 阶段 (Phase) | 视觉反馈内容 |
| :--- | :--- |
| `recording` | 显示实时音频波形图、录音时长。 |
| `transcribing` | 显示“正在转录...”中文提示，进度条呈循环滚动状态。 |
| `polishing` | 显示“正在后处理...”中文提示，强调 AI 处理阶段。 |
| `success` | 显示“成功”标识，并根据配置自动将内容注入系统或剪贴板。 |

## 键盘交互：Esc 键支持

为了提升专业用户的操作效率，系统通过 `NSEvent` 实现了对 **Esc 键 (KeyCode 53)** 的全局支持。

1.  **分流处理**：
    *   **录音中**：按下 Esc 将立即停止采集并**丢弃**当前音频碎片，HUD 直接重置。
    *   **处理中 (转录/润色)**：按下 Esc 将立即隐藏 HUD，但后台任务会保持运行以确保数据不丢失。
2.  **实现逻辑**：`SpeechViewModel` 维护了 `escMonitor`（本地监听）和 `globalEscMonitor`（全局监听），仅在非 Idle 状态下激活，以减少对系统性能的占用。

## 布局稳定性

*   **ZStack 层级**：HUD 使用 `ZStack` 结构，确保在背景列表滚动或窗口缩放时，其位置与渲染不会产生抖动。
*   **响应式适配**：HUD 采用自适应宽度设计，根据提示内容的长度动态调整背景容器大小，同时保持视觉重心居中。
