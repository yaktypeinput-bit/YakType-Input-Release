# macOS 全局快捷键 (Shortcuts)

## 概要说明

本文档详述了 YakType 支持的所有全局与本地快捷键。系统通过 `ShortcutManager` 封装了 `KeyboardShortcuts` 框架，实现了高性能的全局热键监听。用户可在不离开当前工作窗口的情况下，通过这些按键触发复杂的语音处理流水线。

## 核心任务快捷键 (Task Shortcuts)

系统支持为不同的“引擎角色”和“预设任务”分配独立的全局快捷键。

| 快捷键类型 | 触发逻辑 | 行为说明 |
| :--- | :--- | :--- |
| **听写 (Dictation Only)** | 切换触发 (Toggle) | **绕过 AI 润色**。仅执行音频采集与基础 ASR 转换，采集结束后直接注入原始文本。 |
| **主润色 (Main Polishing)** | 切换触发 (Toggle) | **执行完整流水线**。录音结束后，强制调用默认的 Prompt 模板对转录稿进行优化后录入。 |
| **专属任务 (Custom Task)** | 切换触发 (Toggle) | **上下文锁定**。基于特定的 Prompt（如翻译、总结）执行流水线，不受全局默认配置影响。 |

## 录音交互模式 (Interaction Modes)

除了标准的热键触发，系统还针对不同输入场景提供了特定的交互模式：

### 1. 切换模式 (Toggle)
*   **交互逻辑**：按下第一次开始录音，按下第二次停止并提交处理。
*   **适用场景**：长篇大论的构思、会议纪要产出。

### 2. 对讲机模式 (Push-to-Talk)
*   **交互逻辑**：**按住即录音，松开即提交**。此模式绕过了系统的标准热键循环，利用底层事件监听极大降低了交互延迟。
*   **适用场景**：短促的指令输入、即时通讯聊天。默认建议绑定至 `Option + W`。

### 3. 取消操作 (Cancel)
*   **触发方式**：`Esc` 键。
*   **监听优先级**：HUD 处于激活状态时，系统会优先拦截此本地指令。
*   **行为**：立即中断当前的音频流采集或 AI 引擎请求，清理内存缓存并隐藏悬浮窗。

## 快捷键注册机制 (Key Registration)

### 1. 自动重装 (Hot-Reload)
*   **逻辑驱动**：`ShortcutManager` 实时订阅 `EngineProfile` 的数据变更。
*   **一致性保障**：在设置界面修改快捷键组合后，系统会原子化地执行 `unregisterAll()` 并根据最新的 SwiftData 镜像重新加载处理器，无需重启应用。

### 2. 冲突检测与反馈
*   **防撞机制**：如果其他应用先于 YakType 注册了同名全局热键，注册将静默失败。
*   **排查对策**：建议使用 `Option` 或 `Fn` 作为修饰键，避免与 macOS 系统默认的 `Command` 组合产生指令死锁。

## 常见问题处理

| 现象 | 可能原因 | 处理对策 |
| :--- | :--- | :--- |
| **快捷键完全失效** | 辅助功能权限未授予。 | 检查“系统设置 -> 隐私与安全性 -> 辅助功能”。 |
| **按键后 HUD 无反应** | 对应的引擎实例被禁用。 | 进入“执行引擎”面板确认对应 Profile 的 `isEnabled` 状态。 |
| **PTT 模式松开不停止** | 焦点窗口拦截了辅助功能事件。 | 尝试点击一下桌面空白处重新获取交互焦点，或检查是否开启了其他高优先级钩子程序。 |
