# YakType iOS 规约：“听写即输入法”

## 1. 产品愿景
iOS 版 YakType 通过将高性能转录和 AI 润色直接集成到系统键盘中，彻底改变了专业的听写体验。它专为需要直接在任何应用中输入长篇内容、电子邮件或消息的用户设计，兼具多种高性能引擎的准确性与原生输入法的便利性。

---

## 2. 平台架构

### 2.1 多目标策略
- **宿主 App (Host App)**：用于引擎配置、API 密钥管理和提示词自定义。
- **键盘扩展 (Keyboard Extension)**：主要交互界面。包含音频采集管线和转录逻辑。
- **共享框架 (Shared Framework)**：包含 `Sources/Shared` 核心（AudioManager、引擎、模型）的共享目标。

### 2.2 数据共享
- **App Groups**：在宿主 App 和键盘扩展之间共享 `UserDefaults` (AppConfig) 和 `SwiftData` 存储的关键。
- **标识符**：建议使用 `group.com.yaktype.yak-mac` (与 macOS 尽可能保持命名对齐)。

---

## 3. UI/UX 设计（iOS “液态玻璃”）

### 3.1 键盘扩展布局
激活时，键盘扩展将替换标准键盘为专门的“听写仪表盘”。

| 组件 | 描述 |
| :--- | :--- |
| **录制按钮** | 位于中心的大型“脉冲”按钮，带有宇宙脉冲（深紫色）光晕。点击或长按开始录制。 |
| **实时转录区** | 录制按钮上方的可滚动文本区域，展示实时反馈。 |
| **快捷指令区** | 展示由用户在主 App 预设的 **最多三个常用设置**（如特定的引擎组合或常用 Prompt）。 |
| **AI 动作/撤销** | 提供“一键润色”以及在润色后的 **“撤销 (Undo)”** 按钮，点击恢复原始转录。 |
| **引擎切换器** | 键盘上的快捷切换按钮，点击可快速在配置好的 ASR 引擎间轮转或弹出选择菜单。 |
| **状态指示器** | 集成在背景中的波形图或音量计。 |
| **切换键盘** | 返回系统键盘的标准按钮。 |

### 3.2 视觉语言
- **通透感 (Vibrancy)**：使用 `UIBlurEffect` 配合 `.systemThinMaterial` 或 `.systemUltraThinMaterial` 实现“液态玻璃”效果。
- **动画**：录制按钮的和谐缩放；平滑的文本插入动画。
- **触感 (Haptics)**：
    - 开始时触发 `impactOccurred(.medium)`。
    - AI 润色完成时触发 `notificationOccurred(.success)`。

### 3.4 引导流程 (Onboarding)
- **引导页设计**：在宿主 App 首次启动时，提供 3-4 页的快速引导。
- **权限解释**：专门设计一页解释“允许全访问 (Full Access)”的必要性。
    - **内容**：明确告知用户该权限是 **Gemini、阿里云 ASR 及 OpenAI 兼容 API** 进行网络通信的基础。
    - **透明度**：强调 YakType 仅收集语音数据用于转录，不记录平时的按键内容。
    - **操作指引**：提供一键跳转至系统设置界面的按钮。

### 3.5 中断处理逻辑 (Interrupt Handling)
- **目标 App 切换**：当用户在录音过程中切换当前活动 App（即键盘失去焦点）时，录音应 **立即停止**，并完成当前的转录任务，将文本 **记录并注入** 到之前的输入框中。
- **来电处理**：录音过程中若有电话打入，系统将自动触发中断。YakType 应立即停止处理，确保转录结果被 **保存并注入**。

## 4. 技术实现

### 4.1 音频会话管理：热麦 (Warm Mic) 架构
为了克服键盘扩展无法直接访问麦克风的限制，iOS 版采用 **“宿主占用，键盘触发”** 的二级架构。

- **宿主 App (Host App) - 会话管理**：
    - **职责**：作为麦克风的“持有者”，负责 `AVAudioSession` 的激活与锁定。
    - **时长选项**：提供 **“仅本次 / 5 分钟 / 1 小时 / 12 小时”** 的占用时长选择。
    - **锁定机制**：点击“开始占用”后，系统录音图标亮起，麦克风被 YakType 独占，防止被其他 App 抢占会话。此时 **不产生数据流量，不开启 ASR 引擎**。

- **键盘扩展 (Keyboard Extension) - 录音触发**：
    - **职责**：作为数据流的“开关”。
    - **状态流转**：
        - **点击开始**：通过 IPC (App Groups/Notifications) 告知宿主 App 开启 ASR 存储与发送流程（产生流量）。
        - **点击结束**：通知宿主 App 停止 ASR 引擎。此时麦克风 **依然保持被 YakType 锁定**，直到宿主 App 设定的总时长结束。

### 4.2 音频管线与数据传输
- **框架**：`AVFoundation` (AVAudioEngine) 在主 App 台运行。
- **背景模式**：必须启用 `Audio, AirPlay, and Picture in Picture` 后台权限。
- **内存优化**：键盘端仅作为 UI 代理，重量级的转录与 AI 处理逻辑由宿主 App 承载。

### 4.3 文本注入 (Text Injection)
- **机制**：使用 `textDocumentProxy` 协议。
- **逻辑**：
    1.  **实时转录**：宿主 App 将识别结果写入 App Group 共享缓冲区，键盘扩展实时同步并调用 `insertText(_:)`。
    2.  **AI 替换**：获取当前输入的 Context。在进行 AI 润色替换时，先执行 `deleteBackward()` 移除原始文本，再使用 `insertText(_:)` 插入润色后的结果。

### 4.3 引擎集成 (Engine Core)
首个版本仅支持核心两种语言：**中文 (zh-CN)** 与 **英文 (en-US)**。
- **Apple 原生 (Apple Native)**：低延迟、离线、零电池损耗。主要用于基础转录。
- **Gemini / OpenAI 兼容 API**：用于高精度转录和所有“润色”任务。支持自定义服务端地址。
- **阿里云 ASR**：针对中文环境优化的流式识别引擎，适用于高质量专业转录。

## 5. 安全与隐私
- **全访问权限 (Full Access)**：
    - **核心要求**：为了确保云端引擎（Gemini、阿里云及 OpenAI 兼容 API）的连通性，键盘必须开启“允许全访问”。
    - **不设离线降级**：不再提供独立的“隐私优先/离线”模式，产品体验以高精度云端转录为主。
    - **数据最小化**：仅在激活录制时采集并发送音频数据，键盘在空闲时不监听、不记录、不上传任何按键内容。

---

## 6. 实施路线图
1.  **工程骨架**：配置 XcodeGen iOS 目标、设置 App Group。
2.  **UI 开发**：实现键盘扩展的液态玻璃界面。
3.  **核心迁移**：适配 `AudioManager` 至 iOS 平台。
4.  **注入逻辑**：实现 `textDocumentProxy` 相关的文本管理。
5.  **引擎调优**：在 iOS 环境下测试四种引擎的响应速度与内存表现。
